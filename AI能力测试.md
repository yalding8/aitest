
## 调研
市面上是否有AI应用能力的通用考试？

## 核心需求
1. 根据大模型的进度定期更新
2. 考试对象：非技术岗新入职员工（**咨询顾问 + 渠道BD + KA岗位**）
3. 试题库支持随机出题，多题型混合
4. 考试目的：判断候选人是否是**AI狂热分子**

## 产品规格
- **考试形式**：线上考试系统（可分享链接）
- **题量**：30道题目
- **及格线**：80分及以上
- **题库分层**：不按岗位细分（通用AI能力）
- **题型**：多类型（选择题、案例分析等）

## 产品设计确认

### ✅ 已完成

#### 第一阶段：内容准备
- [x] **题库**：60道题，分4个维度均匀分布
  - AI工具认知：15道（单选为主）
  - 岗位场景应用：20道（单选+案例）
  - 思维转变识别：15道（多选+案例）
  - 实战判断：10道（单选+多选）
- [x] **授权码**：50个独立一次性码，格式 EXAM2601xx
- [x] **计分体系**：100分制，每题3-5分，80分及格

#### 第二阶段：前端开发
- [x] **项目架构**：React 18 + TypeScript + Tailwind + Vite
- [x] **身份验证页**：授权码 + 个人信息（姓名、邮箱、岗位）
- [x] **考试页面**：
  - 一题一显示模式（更专注）
  - 60分钟倒计时，自动提交
  - 题目导航面板，快速跳转
  - 实时进度条和答题状态
- [x] **完成页面**：无分数反馈（保护隐私）
- [x] **UI设计**：Apple风格，简洁高质感

#### 第三阶段：后端开发
- [x] **API接口**：
  - `POST /api/verify` - 验证授权码
  - `POST /api/start` - 启动考试，随机抽30道题
  - `POST /api/submit` - 提交答卷，计分和推送
- [x] **核心逻辑**：
  - 随机出题：按维度分层均匀分布
  - 计分算法：对标答案自动计分
  - 授权码管理：一次性使用，自动标记
  - Webhook推送：飞书/钉钉机器人集成
- [x] **技术栈**：Node.js + Express + 轻量级无数据库

#### 第四阶段：部署就绪
- [x] **本地开发环境**：完整配置文件和启动脚本
- [x] **部署选项**：Vercel、Railway、Docker等
- [x] **文档完整**：README + QUICK_START 指南
